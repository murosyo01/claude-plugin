# システム設計チェックリスト

設計提案を確定する前に使用し、すべての要件カテゴリが網羅されているか確認する。

---

## 機能要件

- [ ] ユーザーストーリーまたはユースケースが文書化されている
- [ ] API契約が定義されている（エンドポイント・リクエスト/レスポンスのスキーマ）
- [ ] データモデルが仕様化されている
- [ ] UI/UXフローがマッピングされている（該当する場合）
- [ ] エッジケースとエラーシナリオが特定されている

## 非機能要件

- [ ] パフォーマンス目標が定義されている（レイテンシ p50/p95/p99・スループット RPS）
- [ ] スケーラビリティ要件が仕様化されている（ユーザー数・データ量の成長予測）
- [ ] セキュリティ要件が特定されている（認証・認可・データの機密性）
- [ ] 可用性目標が設定されている（稼働率 %・許容するダウンタイム）
- [ ] データ保持とコンプライアンス要件が確認されている

## 技術設計

- [ ] アーキテクチャ図またはコンポーネント説明が作成されている
- [ ] コンポーネントの責務が定義されている（各モジュールが担うこと）
- [ ] データフローが文書化されている（データの入力・変換・出力）
- [ ] 連携ポイントが特定されている（外部API・サービス・キュー）
- [ ] エラーハンドリング戦略が定義されている（リトライ・フォールバック・サーキットブレーカー）
- [ ] テスト戦略が計画されている（ユニット・インテグレーション・E2Eのスコープ）
- [ ] DBマイグレーション戦略が検討されている（後方互換性のある変更か）

## 運用

- [ ] デプロイ戦略が定義されている（ローリング・ブルーグリーン・カナリア）
- [ ] モニタリングとアラートが計画されている（主要メトリクス・アラートの閾値）
- [ ] ログ戦略が定義されている（何をログするか・ログレベル・保持期間）
- [ ] バックアップと復旧戦略が文書化されている
- [ ] ロールバック計画が文書化されている（デプロイが失敗したときの戻し方）
- [ ] ランブックが更新または作成されている

---

## 小規模機能向け簡易チェックリスト

小さな変更の場合はこちらを使用する:

- [ ] 入力と出力が定義されている
- [ ] エラーケースが処理されている
- [ ] パフォーマンスへの影響が評価されている
- [ ] セキュリティへの影響がレビューされている
- [ ] テスタビリティが考慮されている
- [ ] ロールバックが可能である
