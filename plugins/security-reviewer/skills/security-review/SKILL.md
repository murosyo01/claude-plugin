# セキュリティレビュースキル

git diffまたは指定ファイルを読み込み、セキュリティ脆弱性を専門的に検出して報告する。重大度別（CRITICAL / HIGH / MEDIUM / LOW）に指摘を分類し、APPROVE / WARNING / BLOCK の判定を返す。

## レビュープロセス

### 1. コンテキストの収集

`git diff --staged` と `git diff` を実行してすべての変更を確認する。差分がない場合は `git log --oneline -5` で最近のコミットを確認しファイルを直接読み込む。引数でファイルが指定された場合はそのファイルを優先する。

### 2. 初期スキャン

```bash
# 依存関係の脆弱性チェック（package.json が存在する場合）
npm audit --audit-level=high

# ハードコードされたシークレットを検索
grep -rE "(api_key|apikey|secret|password|token|credential)\s*=\s*['\"][^'\"]{8,}" \
  --include="*.ts" --include="*.js" --include="*.env" .
```

### 3. 高リスクエリアの優先確認

認証・APIエンドポイント・DBクエリ・ファイルアップロード・決済・Webhookのコードを優先して読み込む。

### 4. セキュリティチェックリスト

**CRITICAL（必ずフラグを立てる）**
- ソースコードにハードコードされた認証情報（APIキー・パスワード・トークン）
- SQLインジェクション：パラメータ化クエリではなく文字列連結
- XSS脆弱性：HTML/JSXでエスケープされていないユーザー入力
- パストラバーサル：サニタイズなしのユーザー制御ファイルパス
- 認証バイパス：保護されたルートでの認証チェック欠落
- ユーザー入力付きのシェルコマンド実行
- 平文パスワード比較

**HIGH（要確認）**
- SSRF：ユーザー提供URLへの直接フェッチ（ホワイトリスト不在）
- CSRF脆弱性：状態変更エンドポイントの保護欠落
- ロックなし残高チェック（競合状態）
- レート制限欠落の公開エンドポイント
- 脆弱な依存関係（`npm audit` で検出）
- タイムアウトなし外部HTTP呼び出し

**MEDIUM（注意）**
- ログへの機密情報（パスワード・トークン・PII）漏洩
- エラーメッセージで内部詳細をクライアントに送信
- セキュリティヘッダーの欠落

**LOW（参考）**
- HTTPS未強制
- セキュリティイベントの不十分なログ記録

### 5. 確信度ベースのフィルタリング

指摘を書く前に以下4点を確認する。「いいえ」または「不明」があれば重大度を下げるか削除する:

1. 正確な行を引用できるか（ファイル名と行番号）
2. 具体的な失敗シナリオを説明できるか（入力・状態・悪い結果）
3. 周辺のコンテキスト（呼び出し元・既存バリデーション）を読んだか
4. 重大度は正当か

**確信度80%未満の指摘はスキップする。**

よくある誤検知（スキップ）: `.env.example` 内のシークレット・テストファイルの `test` マーク付き認証情報・設計上公開のAPIキー・チェックサム用SHA256/MD5。

### 6. 結果の報告

各指摘を以下の形式で報告する:

```
[重大度] 問題の概要
ファイル: path/to/file.ts:行番号
問題: 具体的な問題の説明と失敗シナリオ
修正: 推奨される修正方法とコード例
```

すべてのレビューをサマリー（重大度別件数テーブル + APPROVE/WARNING/BLOCK 判定）で締めくくる。
