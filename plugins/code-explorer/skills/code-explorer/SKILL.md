# コード探索スキル

既存コードベースの機能・エリアを深く分析し、実行パスの追跡・アーキテクチャ層のマッピング・依存関係の文書化を行うスキル。新しい開発を始める前の調査に特化し、ファイルの作成・変更は一切行わない。

## 探索プロセス

### 1. エントリポイントの発見

Glob・Read・LSP を使って調査を開始する:

- `Glob` でディレクトリ構造を把握し、分析対象の主要ファイルを特定する
- `LSP documentSymbols` でファイルのシンボル一覧（関数・クラス・エクスポート）を取得する
- `LSP hover` でエントリポイントの型シグネチャとドキュメントを確認する
- ユーザーのアクションまたは外部トリガーの起点を特定する

### 2. 実行パスの追跡

LSP を中心に、Read・Grep で補完する:

- `LSP definition` で関数・メソッドの定義元にジャンプして実装を読む
- `LSP hover` で型情報・戻り値・パラメータを確認しながら呼び出しチェーンを下る
- `LSP findReferences` で特定シンボルがどこから呼ばれているかを逆引きする
- 分岐ロジック・非同期境界・エラーパスを記録する

### 3. アーキテクチャ層のマッピング

LSP・Read・Grep を使って:

- コードが触れる層（ルーティング・サービス・リポジトリ・ドメイン・インフラ）を特定する
- `LSP findReferences` で層間のインターフェース境界と通信パターンを確認する
- 再利用可能な境界と問題のある結合（密結合・循環依存）を記録する

### 4. パターン認識と依存関係の文書化

- `Grep` で命名規則・ファイル構成のパターンを検索する
- `LSP hover` でインポートされた型・関数のシグネチャと出所を確認する
- `LSP findReferences` で内部モジュール間の依存グラフをマッピングする
- 再利用可能なユーティリティ・共有コンポーネントを特定する

## LSP 操作リファレンス

探索で使う主要な LSP 操作:

| 操作 | 用途 | 典型的な使いどころ |
|------|------|------------------|
| `documentSymbols` | ファイル内の全シンボル一覧 | エントリポイント発見・構造把握 |
| `definition` | 定義元へジャンプ | 実行パス追跡・実装確認 |
| `hover` | 型情報・ドキュメント取得 | シグネチャ確認・戻り値の型把握 |
| `findReferences` | 参照箇所の全列挙 | 依存関係マッピング・影響範囲調査 |

詳細な使い方とサンプルは `references/lsp-exploration-guide.md` を参照する。
